# 数据注入渲染方案（v2 架构）

## 核心思路

不叠加渲染子地图，而是在泰南原生渲染管线的**数据获取阶段**注入楼层内容。每个格子只画逻辑上的最高层。

## 当前架构（叠加渲染）的问题

- 需要 `Graphics.DrawMesh` 手动叠加子地图 mesh
- render queue hack + Y offset 处理遮挡，容易出问题
- 光照、阴影、雾、雪不自然（子地图内容不走原生管线）
- 大量补丁处理交互重定向

## 泰南渲染数据流

### 地形 — `SectionLayer_Terrain.Regenerate()`

```csharp
// Verse/SectionLayer_Terrain.cs:53
TerrainGrid terrainGrid = base.Map.terrainGrid;
TerrainDef terrainDef = terrainGrid.TerrainAt(intVec);  // ← 注入点
CellTerrain cellTerrain = new CellTerrain(terrainGrid.TerrainAt(intVec), ...);
```

逐格子从 `terrainGrid` 读地形 → 构建 quad mesh。

### 建筑/物品 — `SectionLayer_Things.Regenerate()`

```csharp
// Verse/SectionLayer_Things.cs:58
List<Thing> list = base.Map.thingGrid.ThingsListAt(intVec);  // ← 注入点
for (int i = 0; i < count; i++)
{
    Thing thing = list[i];
    // ... 过滤条件 ...
    this.TakePrintFrom(thing);
}
```

逐格子从 `thingGrid` 读物体列表 → 调用 `TakePrintFrom` 烘焙进 mesh。

### 动态物体 — `DynamicDrawManager`

逐个 Thing 调用 `DynamicDrawPhase`，已经是格子级别控制。

## 实现方案

### 1. 地形注入

Patch `TerrainGrid.TerrainAt(IntVec3)`：

```csharp
internal static bool IsCompositing = false;

[HarmonyPatch(typeof(TerrainGrid), "TerrainAt", typeof(IntVec3))]
public static class Patch_TerrainGrid_TerrainAt
{
    public static bool Prefix(TerrainGrid __instance, IntVec3 c, ref TerrainDef __result)
    {
        if (!IsCompositing) return true;
        // 只在基地图的 terrainGrid 上生效
        // 查找该格子的最高层
        var topLevel = LevelManager.GetTopmostLevelAt(c);
        if (topLevel?.LevelMap == null) return true;
        __result = topLevel.LevelMap.terrainGrid.TerrainAt(c);
        return false;
    }
}
```

在 section 重建前设 `IsCompositing = true`，重建后设 `false`。

### 2. 建筑/物品注入

Patch `SectionLayer_Things.Regenerate()`，在原版遍历后追加子地图物体：

```csharp
[HarmonyPatch(typeof(SectionLayer_Things), "Regenerate")]
public static class Patch_SectionLayer_Things_Regenerate
{
    public static void Postfix(SectionLayer_Things __instance)
    {
        // 对 section 内每个格子，查找最高层
        foreach (IntVec3 cell in __instance.section.CellRect)
        {
            var topLevel = LevelManager.GetTopmostLevelAt(cell);
            if (topLevel?.LevelMap == null) continue;

            // 从子地图的 thingGrid 获取物体并 Print
            var things = topLevel.LevelMap.thingGrid.ThingsListAt(cell);
            foreach (var thing in things)
            {
                // 需要处理 thing.Map != base.Map 的兼容性
                thing.Print(__instance);
            }
        }
    }
}
```

### 3. 动态物体

保持现有方案或类似注入：在基地图的 `DynamicDrawManager` 绘制时，追加子地图的动态物体。

## 风险点

| 风险 | 说明 | 缓解方案 |
|------|------|----------|
| `TerrainAt` 被大量系统调用 | 寻路、AI、温度等都调用 | 用 `IsCompositing` flag 限定只在渲染期间生效 |
| `thing.Map` 指向子地图 | `Print()` 内部可能访问 `thing.Map` 获取数据 | 需要逐个排查；或临时重定向 Map 引用 |
| `terrainGrid.ColorAt` 等关联调用 | 地形不只是 TerrainDef，还有颜色、污染等 | 需要同时 patch `ColorAt`、`IsPolluted` 等 |
| Section dirty 同步 | 子地图内容变化时需要 dirty 基地图对应 section | 监听子地图的 `MapMeshDirty` 事件 |

## 优势

- 用泰南原生渲染管线，不需要 `Graphics.DrawMesh`
- 不需要 render queue hack、Y offset、ZTest
- 光照、阴影、雾、雪自然生效
- 遮挡问题天然消失——每个格子只有一份数据
- 代码更简洁，维护成本低

## 与当前方案的关系

当前方案（叠加渲染 + TakePrintFrom 补丁）已经可以工作。数据注入方案作为 v2 架构升级，在稳定后考虑迁移。两个方案可以逐步过渡：先迁移地形，再迁移建筑，最后迁移动态物体。

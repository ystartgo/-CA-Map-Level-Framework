# MLF 优化清单

## 一、地图 Tick 抑制（影响最大）

每个层级子地图都是完整 Map，每 tick 跑全套 MapPreTick + MapPostTick。
3 层楼 = 2 个子地图 = 地图系统开销 ×3。

### MapPreTick 中需要跳过的：

| 系统 | 处理方式 | 原因 |
|------|----------|------|
| `windManager.WindManagerTick()` | **跳过** | 楼层内无风 |
| `autoBuildRoofAreaSetter` | **跳过**（地上层） | 楼层屋顶由 MLF 管理 |
| `itemAvailability.Tick()` | 保留 | 物品可用性追踪需要 |
| `listerHaulables.ListerHaulablesTick()` | 保留 | 搬运系统需要 |
| `roofCollapseBufferResolver` | 保留 | 结构完整性 |
| `mapTemperature.MapTemperatureTick()` | 保留 | 温度计算需要 |
| `pathFinder.PathFinderTick()` | 保留 | 寻路需要 |
| `temporaryThingDrawer.Tick()` | 保留 | 视觉效果 |

### MapPostTick 中需要跳过的：

| 系统 | 处理方式 | 原因 |
|------|----------|------|
| `wildAnimalSpawner.WildAnimalSpawnerTick()` | **跳过** | 楼层不刷野生动物 |
| `wildPlantSpawner.WildPlantSpawnerTick()` | **跳过** | 楼层不刷野生植物 |
| `passingShipManager.PassingShipManagerTick()` | **跳过** | 子地图无过路商船 |
| `weatherManager.WeatherManagerTick()` | **跳过** | 已与主地图共享 weatherManager |
| `weatherDecider.WeatherDeciderTick()` | **跳过** | 已与主地图共享 weatherDecider |
| `gameConditionManager.GameConditionManagerTick()` | **跳过** | 已与主地图共享 |
| `lordsStarter.VoluntarilyJoinableLordsStarterTick()` | **跳过** | 子地图不独立发起聚会 |
| `pollutionGrid.PollutionTick()` | **跳过**（地上层） | 楼层无污染扩散 |
| `waterBodyTracker.Tick()` | **跳过** | 楼层无水体 |
| `TileMutatorDef.Worker.Tick()` | **跳过** | 世界地块变异不适用子地图 |
| `powerNetManager.PowerNetsTick()` | 保留 | 电网需要 |
| `steadyEnvironmentEffects.SteadyEnvironmentEffectsTick()` | 保留 | 物品腐烂/风化需要 |
| `gasGrid.Tick()` | 保留 | 气体扩散需要 |
| `lordManager.LordManagerTick()` | 保留 | 袭击 AI 需要 |
| `fireWatcher.FireWatcherTick()` | 保留 | 火灾监控需要 |
| `resourceCounter.ResourceCounterTick()` | 保留 | 资源统计需要 |
| `flecks.FleckManagerTick()` | 保留 | 视觉效果 |
| `effecterMaintainer.EffecterMaintainerTick()` | 保留 | 视觉效果 |
| `deferredSpawner.DeferredSpawnerTick()` | 保留 | 延迟生成 |
| `MapComponentUtility.MapComponentTick()` | 保留 | 自定义组件 |

### 实现方式：
Harmony Prefix patch `MapPreTick` 和 `MapPostTick`，检测 `LevelMapParent` 后手动调用需要保留的系统，return false 跳过原方法。

## 二、Tick 格子过滤（可用区域限制）

层级子地图是全尺寸 Map（如 250×250 = 62500 格），但实际可用区域可能只有一小块（如 30×30 = 900 格）。
寻路（A*）自动跳过不可通行格子，但其他按格子遍历的系统不会，白白浪费大量计算。

| 系统 | 问题 | 方案 |
|------|------|------|
| `steadyEnvironmentEffects` | 随机选格子检查腐烂/风化，大部分选到空白区域白跑 | Patch `SteadyEnvironmentEffectsTick`，对 level map 限制随机选格范围到 `usableCells` |
| `regionGrid` / `regionAndRoomUpdater` | 为整张地图构建 region，空白区域也建 | 考虑在地图生成时将非可用区域标记为已完成的 region，减少重建开销 |
| `mapTemperature` | 温度均衡遍历 room，空白区域的巨大 room 拖慢计算 | 确保非可用区域不形成巨大 room（用 impassable 地形隔断） |
| `glowGrid` | 光照计算覆盖全图 | 非可用区域无光源，开销较小，暂不处理 |
| `pathFinder` | A* 自动跳过不可通行格子 | **已自动过滤，无需处理** |
| `gasGrid` | 气体只在有气体的格子扩散 | **已自动过滤（稀疏存储），无需处理** |
| `fireWatcher` | 只检查有火的格子 | **已自动过滤，无需处理** |

### 关键前提：
确保层级子地图的非可用区域全部是 impassable 地形（如已有的 MLF_OpenAir），这样：
- 寻路自动跳过 ✓
- region 系统不会为空白区域建大量小 region ✓
- 温度系统不会形成覆盖全图的巨大 room ✓
- `steadyEnvironmentEffects` 仍需 patch（它是随机选全图格子，不管 passability）

## 三、热路径代码优化

### 优先级 1：`GetLevelForMap` LINQ 消除（覆盖面最广）

**位置：** `LevelManager.cs:183`
**问题：** `levels.Values.FirstOrDefault(d => d.LevelMap == levelMap)` 每次分配 enumerator + delegate，被 `IsLevelMap` 调用，而 `IsLevelMap` 在几乎所有 patch 中都有。
**方案：** 在 `LevelMapParent` 上缓存 `LevelData` 引用，`IsLevelMap` 直接读取，不再调用 `GetLevelForMap`。

### 优先级 2：`FindStairsToElevation` 全 Thing 扫描（每次跨层扫描触发）

**位置：** `CrossLevelJobUtility.cs:179-200`
**问题：** 遍历 `map.listerThings.AllThings`（可能数千个）找楼梯，且对每个匹配楼梯调用 `CanReach`（寻路开销）。
**方案：** 维护 `Dictionary<Map, Dictionary<int, List<Building_Stairs>>>` 缓存，楼梯 Spawn/DeSpawn 时更新。先按距离排序，只对最近的调用 `CanReach`。

### 优先级 3：`AllLevels.Any()` 枚举器分配（十几处调用）

**位置：** 多处（WorkGiver、CrossLevelJobUtility、JobGiver_Assault 等）
**问题：** `Dictionary.Values.Any()` 每次分配 enumerator。
**方案：** 添加 `LevelManager.LevelCount` 属性，全部改为 `mgr.LevelCount > 0`。

### 优先级 4：`TryCrossLevelScan` 反射 + 分配

**位置：** `CrossLevelJobUtility.cs:61-163`
**问题：**
- `new List<(Map, int)>()` 每次分配
- `FieldInfo.GetValue/SetValue` 装箱开销
- `Find.Maps.IndexOf` O(n) 查找
**方案：**
- static list 复用（clear + reuse）
- `AccessTools.FieldRefAccess` 替代 FieldInfo 反射
- 在 LevelData 上缓存 map index

### 优先级 5：需求 patch 的 `MethodInfo.Invoke`（7 处）

**位置：** `Patch_CrossLevelNeeds.cs` 全部 7 个 patch
**问题：** `method.Invoke(__instance, new object[] { pawn })` 每次分配 object[]，反射调用慢。
**方案：** 用 `method.CreateDelegate<Func<JobGiver_xxx, Pawn, Job>>()` 编译为委托。

### 优先级 6：渲染路径每帧分配

**位置：** `Patch_MapDrawer.cs:16, 32-45`
**问题：**
- `Traverse.Create(__instance)` 每帧分配 Traverse 对象
- `new HashSet<IntVec3>()` 每帧分配并填充数千 cell
**方案：**
- Traverse 换成 cached `AccessTools.FieldRefAccess`
- `excludeCells` 缓存到 LevelData，仅在 usableCells 变化时重建

### 优先级 7：`Find.Maps.IndexOf` 多处 O(n)

**位置：** `Patch_Overlays.cs:44`、`Patch_FloatMenuContext.cs:37`、`Patch_Selector.cs:99` 等 6+ 处
**问题：** 每帧多次 O(n) 线性查找。
**方案：** 在 LevelData/LevelMapParent 上缓存 mapIndex，地图增删时更新。

### 优先级 8：`LevelRenderer` 反射 + 字符串分配

**位置：** `LevelRenderer.cs:185-203`
**问题：**
- `FieldInfo.GetValue` 每 section 每帧
- `layer.GetType().Name` 每 layer 每帧分配字符串
**方案：**
- `AccessTools.FieldRefAccess` 替代
- `typeof()` 比较替代字符串比较，或缓存 `HashSet<Type>` skip 列表

### 优先级 9：`Patch_RoofFloorSync` 反射

**位置：** `Patch_RoofFloorSync.cs:225-249`
**问题：** 每次 SetRoof/SetTerrain 调用 `FieldInfo.GetValue`。
**方案：** 用 Harmony `___map` 参数注入替代手动反射。

### 优先级 10：`Patch_Thing_Destroy` 全 Thing 拦截

**位置：** `Patch_Mineable_Destroy.cs:13`
**问题：** patch 了 `Thing.Destroy`，所有 Thing 销毁都触发，tuple state 每次分配。
**方案：** 改为 patch `Mineable.Destroy`（如果存在），或在 prefix 最前面加 `if (!(__instance is Mineable)) return` 并用 `__state = default` 避免分配。
